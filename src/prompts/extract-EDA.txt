
コンテンツについて議論した論文がUser Messageとして与えられるため、そのEDAを解析してください。

# EDAとは何か？
コンテンツのEntertainment Design Assets (EDA) とは、そのコンテンツのプレイヤに対して与える「心の動き」を説明した文言を意味します。

## EDA アイテム群の具体例
「実世界とのリンク」「ナラティブな体験」
「『魅せる』演出」「人に語りたくなる要素」
「直感的なインターフェース」「費用対効果のジレンマ」
「自己表現な可能な余地」「達成のための共同作業」
「段階的な学習」「開放感」「羞恥心」「背徳感」「ナラティブ」
「操作感 -> 達成感」「ボールを転がすことによるそこはかとなさ」
「Webサイト，Webページの占有欲」
「動きと視聴覚の同期性」
この際、
- 何をすることによってどのような感情を感じられるか
- 驚き
- どのようなEDAが生み出されるかを明示的に説明するように、修飾したEDAがより望ましい。
e.g. 「お掃除して部屋がピカピカになることによる気持ちよさ」

    “EDA Conditions”:
    - "The feeling responses targeted in the user are clearly defined as part of the design requirements of the content."
        - It is important that they are specifically and unambiguously defined by the developers.
        - If the feeling experience varies based on user behavior (i.e., the EDA becomes optional), this condition is not satisfied.
    - "The content aims to deliberately influence the user's own feeling state."
    - "The paper discusses what kind of feeling response the content is expected to evoke when used."


# 出力形式
次のjson形式でコードブロックの内部に出力してください。
ただし、jsonコメントは使用することができない点に注意すること。
「〜による〇〇」という形式で説明するとよい。
```
{
    // 本文中で**確実に**言及されているEDAを抜き出してください。
    // そのコンテンツが巻き起こすと期待されている心の動きのみを書き起こすこと。
    // 数よりも質を重視してください。
    "proposed-EDA": string[], 

    // 本文中から明瞭に推定できるEDAがあればそのようなEDAをここで並べてください。
    // なければ[]で。確信度が高い順に並べましょう。
    // 数よりも質を重視してください。
    "assumed-EDA": string[]
}
```
