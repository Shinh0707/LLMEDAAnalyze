
コンテンツについて議論した論文がUser Messageとして与えられるため、そのEDAを解析してください。

# EDAとは何か？
コンテンツのEntertainment Design Assets (EDA) とは、そのコンテンツのプレイヤに対して与える「心の動き」を説明した文言を意味します。

## EDAの定性的定義
次の条件が満たされている場合に限り、コンテンツ$C$はEntertainment Design Assets$A$を備えている：
I. Defined（明示的定義）
    心の動き$A$がコンテンツ$C$の設計要件として明確に定義・言及されている。
II. Suggested（調和性）
    心の動き$A$がコンテンツ$C$の既存の設計要件・EDAを阻害せず、それらと共存可能である。
III. Contextual Specificity（文脈的特異性）
    心の動き$A$は$C$における設計要件により、
        ① その要件が存在しない場合に比べて$A$の程度が著しく向上している
        または
        ② 要件が存在しない場合とは異なる体験価値が生まれている
    場合に限りEDAとして妥当である。
「〜による〇〇」という形式で説明するとよい。

## EDA アイテム群の具体例
「実世界とのリンク」「ナラティブな体験」
「『魅せる』演出」「人に語りたくなる要素」
「直感的なインターフェース」「費用対効果のジレンマ」
「自己表現な可能な余地」「達成のための共同作業」

# 出力形式
次のjson形式でコードブロックの内部に出力してください。
```
{
    // 本文中で**確実に**定義されたEDAを記述してください。
    // 数よりも質を重視してください。該当するものがなければ空配列[]で問題ありません。
    "defined-by-developer-EDA": string[], 

    // このシステムで恩恵を受ける人（ターゲット）について考察してください。どのようなことを実現したいと考える人でしょうか？
        // Maslowの欲求段階説：生理的欲求・安全の欲求・社会的欲求・承認欲求・自己実現の欲求
    // 網羅性よりも、開発者が想定する主要な人を挙げるように心がけましょう。数よりも質を重視してください。
    // 「〜〜したい〇〇」という形式で書きましょう。
    "target": string[],

    // 上で挙げたターゲット`target`が、実験とは異なる文脈でこのシステムに触れたとします。
    // 実験の記述にとらわれず、その人がこのシステムを使って何をするかを推量しましょう。
    // このシステムはいつ、どこで、どのように、どうやって、何のために使われるでしょうか？
    // できるだけ多くのケースを掲げてください。
    "use-case": string[],

    // `target`が`use-case`を体験した時に生まれるであろうEDAについて考察してください。
    // 数よりも質を重視してください。
    "suggested-EDA": string[],
}
```
