# 指示

Entertainment Design Asset(EDA)とは、心の動きをもたらすデザインを記述したものを表す。
エンタテインメントコンテンツの分類において有用とされている。

EDAはシステムに関する次の詳細な記述から構成される。
- 外部仕様：要件を満たすために、システムがどのように振る舞うか。
- 要件定義：目的を達成するために、システムがどのような条件を満たすべきか。
- ターゲット：エンタテインメントコンテンツを体験するユーザは、どのような属性を持っているか？どのような体験をしたい（欲求）と思っているか？
- 目的・起こす情動：システムがどのような「心の動き」を達成するか。基本、情動語で表す。

良いEDA (good EDA)とは、次の要件を満たすものである。
  	極小性：要件定義・仕様は掲げた「心の動き」を実現する極小のものであると示せる。
  		言い換えれば、要件・仕様を一部欠くシステムでは達成できないことが明確に判別できる。

さて、以下に提示する論文は、エンタテインメントコンテンツについて研究した成果を報告する原稿である。
この論文で提唱されているシステムや議論されている内容をもとに、"良いEDA"として整理し直すことができるだろうか？
整理する際、推測を加えることなく、論文で述べられている内容だけを用いて整理せよ。

# 回答形式

良いEDAとして整理しなおせるのであれば、次のJSON形式で回答を出力せよ。
```json
{
	"EDAの存在": true,
	"外部仕様": string[],
	"要件定義": string[],
	"ターゲット": string[],
	"目的・起こす情動": string[],
}
```

EDAとして整理し直すことができない（情動をもたらすことを目的としてデザインされていないコンテンツ）、
あるいは極小性が守られない場合（良いEDAでないもの）は次のように出力せよ。
```json
{
	"existsの存在": false,
	"外部仕様": string[],
	"要件定義": string[],
	"EDAでない理由": string[]
}
```
