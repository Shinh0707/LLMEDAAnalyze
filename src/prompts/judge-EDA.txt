# 前提知識
コンテンツにおけるEntertainment Design Assets（以下EDA）は、
エンタテインメントコンテンツにおいて心を動かすデザインを説明した文書を指し、
コンテンツを分類するにあたって有用な指標と期待されている。

EDAは、システムに関する次の記述から構成される。
- 目的：システムがどのような「心の動き」を達成するか。
- 要件定義：目的を達成するために、システムがどのような条件を満たすべきか。
- 外部仕様：要件を満たすために、システムがどのように（厳密に）振る舞うべきか。
また、良いEDAとしては、次の要件を満たす必要がある。
- 極小性：要件定義・仕様は掲げた「心の動き」を実現する極小のものであると示せる。
 	- 言い換えれば、要件・仕様を一部満たさないシステムでは達成できないことが明確に判別できる。

ここでいう心の動きの具体例を以下に示そう。
実世界とのリンク / ナラティブな体験
/ 「魅せる」演出 / 人に語りたくなる要素
/ 直感的なインターフェース / 費用対効果のジレンマ
/ 自己表現な可能な余地 / 達成のための共同作業
/ 段階的な学習

# 指示

論文がEDAについて議論しているとは、次のような要件1, 2, 3を全て満たしていることとする。
- あるシステムSystemによって提供されたエンタテインメント的な体験を享受するユーザをUserとする。
	- このSystemは著者らが開発したもののほか、既存のものでも良いが、論文の主題となるシステムでなくてはならない。

- 要件1. SystemをUserが体験することによって引き起されるUserの心の動き・情動がなんであるかを、システム設計の目標として開発者が明示している。
	- 明示の程度については次のとおりである。
	- 目的とする心の動きは「楽しさ」とか「驚き」ぐらいの曖昧なものでも良い。
	- 特定の心の動き・情動を内包的に掲げた上で、それらに絞ってシステムを設計していることが必要である。
	- そのシステムの構成だからこそ発生するものを述べる。

- 要件2. システムの要求事項・外部仕様が、掲げた心の動きに貢献していることが要求事項と仕様の記述から推測できればよい。
	- 必要最小性は論文側で実験的に示される必要はなく、デザインの記述から読み取れれば良い。

- 要件3. システムの目的を満たすためのSの要求事項について解説・議論しており、Sのデザインによって目標に掲げた「心の動き」が達成されるという因果関係が、要求事項・仕様の記述から読み取れる。

それぞれの要件1, 2, 3に対して、当てはまるか、当てはまらないかをtrue / falseで回答せよ。

# 回答形式
次のようなコードブロックの形式で解答せよ。
```json
{
	"Systemによって提供されたエンタテインメント的な体験を享受するユーザ": string, // 明確に記述されていなければ空文字列""を用いよ。これは評価において定められるものではなく、設計段階で想定されていなくてはならない。
	"要件1の理由": string,
	"要件1": boolean,
	"要件2の理由": string,
	"要件2": boolean,
	"要件3の理由": string,
	"要件3": boolean,
}
```
