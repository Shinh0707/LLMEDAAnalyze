# 前提知識
コンテンツにおけるEntertainment Design Assets（以下EDA）は、
エンタテインメントコンテンツにおいて心を動かすデザインを説明した文書を指し、
コンテンツを分類するにあたって有用な指標と期待されている。

EDAは、システムに関する次の記述から構成される。
- 目的：システムがどのような「心の動き」を達成するか。
- 要件定義：目的を達成するために、システムがどのような条件を満たすべきか。
- 外部仕様：要件を満たすために、システムがどのように（厳密に）振る舞うべきか。
また、良いEDAとしては、次の要件を満たす必要がある。
- 極小性：要件定義・仕様は掲げた「心の動き」を実現する極小のものであると示せる。
 	- 言い換えれば、要件・仕様を一部満たさないシステムでは達成できないことが明確に判別できる。

ここでいう心の動きの具体例を以下に示そう。
実世界とのリンク / ナラティブな体験
/ 「魅せる」演出 / 人に語りたくなる要素
/ 直感的なインターフェース / 費用対効果のジレンマ
/ 自己表現な可能な余地 / 達成のための共同作業
/ 段階的な学習

# 指示

論文が EDA（Entertainment Design Asset）について議論しているためには、以下の要件1～3をすべて満たしていることとする。

System と User の定義
- System：論文の主題となる、著者が開発したもの、または既存のエンタテインメント体験提供システム
- User：System によって提供されたエンタテインメント的体験を享受する対象

- 要件1: コンテンツ性
	- Systemは、Userに対して「心の動き」や「情動」を起こすために設計されているか？
- 要件2: 明示性
	- System が引き起こそうとする「心の動き」を列挙し、そのための体験について示しているか？
	- 「楽しさ」「驚き」といった抽象的な表現でも、体験や情動に関する言及があれば十分とみなす。
- 要件3: 目標の固定
	- 開発者がUserに対して感じさせたい心の動きが網羅的に記述されており、ユーザの入力に対して目標が不定ではない。

以上の要件1～3について、該当すれば true、該当しなければ false で回答してください。

# 回答形式
まず、以下の問いに答えよ。
- Q1. 論文で言及されている「心の動き」の集合Tを外延的に全て記述せよ。
- Q2. そのうち、システム開発者が設計思想としてUserに対して起こしたい心の動きを選びだせ。あれば全てを書き出し、なければ、空集合{}を答えよ。
上二つの集合を全て記述できたら、次のようなコードブロックの形式で解答せよ。
```json
{
	"Systemによって提供されたエンタテインメント的な体験を享受するUser": string, // 明確に記述されていなければ空文字列""を用いよ。これは評価において定められるものではなく、設計段階で想定されていなくてはならない。
	"要件1の理由": string,
	"要件1": boolean,

	"要件2の理由": string,
	"要件2": boolean,
	
	"要件3の理由": string,
	"要件3": boolean,
}
```
