# 前提知識
コンテンツにおけるEntertainment Design Assets（以下EDA）は、
エンタテインメントコンテンツにおいて心を動かすデザインを説明した文書を指し、
コンテンツを分類するにあたって有用な指標と期待されている。

ここでいう心の動きの具体例を以下に示そう。
実世界とのリンク / ナラティブな体験 / 「魅せる」演出 / 人に語りたくなる要素 / 直感的なインターフェース / 費用対効果のジレンマ
/ 自己表現な可能な余地 / 達成のための共同作業 / 段階的な学習

# 指示

System と User の定義は次のとおりとする。
- System：論文の主題となる、著者が開発したもの、または既存のエンタテインメント体験提供システム
- User：System によって提供されたエンタテインメント的体験を享受する対象
	- これは評価・実験において定められるものではなく、設計段階で想定される必要がある。

以下を判定せよ。
- 要件1. コンテンツ性
	- SystemはUserに対して「心の動き」や「情動」を起こすことを目標に掲げているか？
- 要件2. 明示性
	- System が引き起こそうとするUserの「心の動き」の内容に言及しているか？
	- 「楽しさ」「驚き」といった抽象的な表現でも十分とみなす。
- 要件3.
	- Userの「心の動き」を達成するため、Systemに必要な要求事項・外部仕様に言及しているか？

以上の要件1 - 3について、該当すれば true、該当しなければ false で回答せよ。

# 回答形式
上二つの集合を含めて、次のようなコードブロックの形式で解答せよ。
```json
{
	"Systemによって提供されたエンタテインメント的な体験を享受するUser": string, // 明確に記述されていなければ空文字列""を用いよ。

	"要件1の理由": string,
	"要件1": boolean,

	"要件2の理由": string,
	"要件2": boolean

	"要件3の理由": string,
	"要件3": boolean
}
```
